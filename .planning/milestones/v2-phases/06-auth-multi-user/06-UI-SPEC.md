## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding within form labels |
| sm | 8px | Gap between form field and its error message; OAuth button icon gap |
| md | 16px | Gap between form fields; horizontal card padding |
| lg | 24px | Gap between form section and OAuth section; card vertical padding |
| xl | 32px | Login card internal padding (top/bottom) |
| 2xl | 48px | Login page vertical centering offset from NavBar |
| 3xl | 64px | Login page top/bottom page padding on desktop |

Exceptions:
- Avatar/initials circle: 32px × 32px (size-8) — fixed square, not from spacing scale
- NavBar height: 56px (h-14) — inherited from existing nav-bar.tsx; do not change
- Touch targets for auth buttons: minimum 44px height — use `h-11` (44px) on the "Sign In" and "Create Account" submit buttons (larger than default button `h-8`)

## Color

All tokens are CSS variables from globals.css. Reference by variable name, not hardcoded hex.

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | `bg-background` | Login page background, full-page surface |
| Secondary (30%) | `bg-card` / `bg-muted` | Login card container; tab toggle inactive state; input background |
| Accent (10%) | `bg-primary` / `text-primary` | Submit buttons only; active tab underline (matching nav-bar.tsx `border-primary` pattern) |
| Destructive | `text-destructive` / `border-destructive` | Inline form validation errors and `aria-invalid` input states only |

Accent reserved for:
1. "Sign In" submit button (primary variant)
2. "Create Account" submit button (primary variant)
3. Active tab underline indicator on the Sign In / Create Account toggle

Do NOT use accent on:
- OAuth buttons (use outline variant)
- "Sign In" NavBar link (use `text-muted-foreground` → `text-foreground` on active, matching existing nav link pattern)
- Avatar circle background (use `bg-muted` with `text-foreground` initials)

Dark mode: globals.css `.dark` tokens apply automatically via Tailwind dark variant — no additional work required.

## Screen: `/login`

**Layout:** Full-page centered layout. NavBar present at top (sticky, 56px). Page body centers the login card vertically with `min-h-[calc(100vh-56px)] flex items-center justify-center`.

**Login card structure (top to bottom):**

1. Heading: "Sign in to SWU Tracker" — `text-xl font-heading font-semibold` (Oxanium, 20px, 600)
2. Tab toggle row: "Sign In" | "Create Account" — full-width, bottom-border indicator style
3. Form fields:
   - Email: `<Input type="email" placeholder="you@example.com" />`
   - Password: `<Input type="password" placeholder="Password" />`
   - (Create Account only) Confirm Password: `<Input type="password" placeholder="Confirm password" />`
4. Inline error message zone: `<p className="text-sm text-destructive">` — renders below the relevant field with `aria-invalid` on the input
5. Submit button: full-width (`w-full`), size `lg` (h-9 → override to h-11 for 44px touch target), variant `default` (primary)
   - Sign In tab: label "Sign In"
   - Create Account tab: label "Create Account"
6. OAuth divider: "or continue with" centered text
7. OAuth buttons: two full-width buttons, variant `outline`, size `lg` (h-11)
   - Google: Lucide `ChromeIcon` (or inline SVG if Lucide lacks brand icon) + "Continue with Google"
   - Discord: inline SVG brand icon + "Continue with Discord"

**Note on OAuth icons:** Lucide does not include brand logos (Google, Discord). Use inline SVG `<svg>` elements for OAuth brand icons. Keep them 20×20px (`size-5`), positioned with `gap-2` alongside button text.

## Screen: NavBar — Auth Slot

**Logged-out state:**
- Right side of NavBar: `<Link href="/login">` styled as `text-sm text-muted-foreground hover:text-foreground transition-colors` with label "Sign In"
- No border-b-2 underline (not a page nav link; it is an action link)

**Logged-in state:**
- Right side of NavBar: `<DropdownMenu>` with trigger = avatar circle
- Avatar circle: `size-8 rounded-full bg-muted flex items-center justify-center text-xs font-semibold text-foreground uppercase select-none cursor-pointer`
- DropdownMenuContent: align `end`, sideOffset 8
- Single item: `<DropdownMenuItem variant="default">Sign Out</DropdownMenuItem>`
  - Clicking Sign Out calls Better Auth's `signOut()` client method then redirects to `/login`
- No additional items — no email display, no settings, no placeholders

## Interaction States

### Form Validation

- Validate on submit, not on blur (reduces noise while typing)
- Show inline error below the specific field that failed
- Apply `aria-invalid="true"` to the input — existing Input component already styles `aria-invalid` with `border-destructive ring-destructive/20`
- Error text: `text-sm text-destructive` immediately below the input, role `alert`

### Loading / Pending State

- Submit button: disabled + shows Lucide `Loader2Icon` spinning with `animate-spin` inside button while request is in flight
- OAuth buttons: same — disabled + spinner on the clicked button only

### Redirect Behavior

- After successful sign-in or account creation: `router.push('/')` — per D-04 from CONTEXT.md
- Collection controls clicked while logged out: `router.push('/login')` — per D-03 from CONTEXT.md
- Accessing `/collection`, `/decks`, `/decks/[id]` without session: server-side redirect to `/login` via Better Auth middleware (proxy.ts)

## Accessibility Requirements

- Login form: `<form>` with `aria-label="Sign in"` (Sign In tab) or `aria-label="Create account"` (Create Account tab)
- All inputs: paired `<label>` elements — do NOT use placeholder-only labeling
- Error messages: `role="alert"` so screen readers announce on inject
- OAuth buttons: `aria-label="Sign in with Google"` / `aria-label="Sign in with Discord"` (icon-only clarity)
- Avatar trigger button: `aria-label="User menu"` — the circle shows only a letter
- Focus ring: already handled globally by `outline-ring/50` in globals.css + button/input focus-visible styles
- Color contrast: OKLCH tokens in globals.css satisfy WCAG AA at default theme — no changes needed

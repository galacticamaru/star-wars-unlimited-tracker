# Phase 10.1: Fix sideboard filter in binder Looking For query - Context

**Gathered:** 2026-05-12
**Status:** Ready for research/planning

<domain>
## Phase Boundary

Phase 10.1 is a targeted fix for a logic bug in the Trade Binder's "Looking For" section. Currently, the query that calculates missing cards from the user's decks (`src/db/queries/binder.ts`) incorrectly includes sideboard cards, leading to inflated "Looking For" quantities. This fix will bring the binder's auto-sync logic in line with the Global Want List (`src/lib/want-list.ts`), which already excludes sideboard cards.

</domain>

<decisions>
## Implementation Decisions

### Logic Alignment
- **D-01: Exclude Sideboard Always.** Sideboard cards will be excluded from the auto-calculated "Looking For" list by default. This ensures the public binder correctly reflects the user's primary trade needs (main deck completion).
- **D-02: Surgical Patch.** The fix will be applied directly to the query in `src/db/queries/binder.ts` rather than performing a larger refactor of the want-list logic. This minimizes risk and maintains the performance of the existing composite query.

### User Control
- **D-03: Manual Overrides Remain.** If a user specifically wants to trade for cards for their sideboard, they can still use the "Manual Wants" feature on the binder management page. This manual entry will override the auto-calculated exclusion.

</decisions>

# Phase 5.2: Rarity Filter Fix

## Goal
Implement the rarity predicate in `filterCards()` so the rarity dropdown in the card catalog actually filters the results. Currently, the rarity filter is hardcoded to `true`, which bypasses the filter regardless of user selection.

## Context
- **Files involved**:
  - `src/lib/filter-cards.ts`: Implementation of the filtering logic.
  - `src/lib/filter-cards.test.ts`: Unit tests for the filtering logic.
  - `src/components/catalog/catalog-client.tsx`: Defines `RARITY_OPTIONS`.
- **Rarity Values**:
  - Database (`card_printings.rarity`): `Common`, `Uncommon`, `Rare`, `Legendary`, `Special`.
  - UI Labels (`RARITY_OPTIONS`): `(C) Common`, `(U) Uncommon`, `(R) Rare`, `(L) Legendary`.
- **Current State**: `matchesRarity` is hardcoded to `true` in `filterCards`.

## Requirements
1. `filterCards` must evaluate `selectedRarities`.
2. If `selectedRarities` is empty, all cards should match (no filtering).
3. If `selectedRarities` has values, cards must match one of the selected rarities.
4. The logic must handle the discrepancy between UI labels (e.g., `(C) Common`) and DB values (e.g., `Common`).
5. Unit tests must be updated to verify the fix.

## Success Criteria
1. Selecting one or more rarities from the rarity dropdown filters the catalog to only show cards of those rarities.
2. Selecting no rarities (cleared filter) shows all cards regardless of rarity.
3. Rarity filter composes correctly with all other active filters.
4. `npm test src/lib/filter-cards.test.ts` passes.

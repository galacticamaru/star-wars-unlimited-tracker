<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| env vars → drizzle-kit | DATABASE_URL read from .env file; must not be committed |
| env vars → application | DATABASE_URL and CRON_SECRET must live in .env.local (never source) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | .env / .env.local | mitigate | Both files listed in .gitignore; .env.example documents required vars without values |
| T-01-02 | Elevation of Privilege | /api/cron/sync-cards (future) | mitigate | CRON_SECRET checked via `Authorization: Bearer` header in cron route (Plan 04) |
| T-01-03 | Tampering | swu-db.com API response → DB | mitigate | All DB writes use Drizzle parameterized queries — no raw SQL string interpolation (Plan 03) |
</threat_model>

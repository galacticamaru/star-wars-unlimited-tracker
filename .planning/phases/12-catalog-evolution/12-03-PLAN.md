<task type="checkpoint:human-verify" gate="blocking">
  <name>Checkpoint: Verify New Catalog Layout and Functionality</name>
  <what-built>
    The new catalog layout has been implemented, featuring a sticky sidebar on desktop and a filter sheet on mobile. The data logic for variant filtering is also in place.
  </what-built>
  <how-to-verify>
    Please perform the following checks on the live preview environment:

    **Desktop (Viewport > 768px):**
    1.  Navigate to the `/cards` page.
    2.  **Verify Stickiness**: Confirm a filter sidebar is visible on the left. Scroll the main card grid up and down; the sidebar should remain fixed in place.
    3.  **Verify Independent Scroll**: If the filter list in the sidebar is long (e.g., expand "Traits"), scroll the sidebar's content. The main card grid should not move.
    4.  **Verify Variant Filter**:
        - By default, only 'Normal' cards should be visible.
        - In the sidebar, find the "Variant" filter and select "Showcase". The grid should update to show only Showcase cards.
        - Select "All". The grid should show a mix of Normal, Showcase, etc.

    **Mobile (Viewport < 768px):**
    1.  Navigate to the `/cards` page on a simulated mobile device.
    2.  **Verify Mobile Bar**: Confirm there is no sidebar. You should see a simplified top bar with a "Refine Results" button (or similar).
    3.  **Verify Sheet**: Tap the "Refine Results" button. A sheet/drawer should slide in from the right, containing all the filter controls.
    4.  **Verify Filtering**: Apply a filter from within the sheet (e.g., select a Set) and close it. The card grid behind it should update accordingly.
  </how-to-verify>
  <resume-signal>
    Type "approved" if all checks pass. If you find any issues, please describe them (e.g., "sidebar is not sticky on scroll", "mobile sheet does not open").

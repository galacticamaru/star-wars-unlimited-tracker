## Layout Contracts

### 1. Top Navigation Bar (`src/app/layout.tsx`)

Source of decision: D-09, D-10 (05-CONTEXT.md)

**Structure:** Full-width bar rendered in `RootLayout`, above `{children}`. Height: 56px. Position: sticky top-0, `z-50`.

Visual style: `bg-background/80 backdrop-blur-sm border-b border-border` — matches the existing catalog top bar for visual consistency.

```
[  Star Wars Unlimited  |  Catalog    Collection    Decks  ]
  (brand mark, left)       (nav links, right or centered)
```

- Brand mark (left): Text "SWU Tracker" in Oxanium 16px semibold, `text-foreground`. Not a link.
- Nav links (right-justified via `ml-auto`): Three links — **Catalog** (`/`), **Collection** (`/collection`), **Decks** (`/decks`).
- Nav link styling (inactive): 14px Nunito Sans weight 400, `text-muted-foreground`, `hover:text-foreground transition-colors`, `px-md py-sm` (16px × 8px), minimum touch target 44px height via `h-11 flex items-center`.
- Nav link styling (active): Same padding, `text-foreground font-semibold`, bottom border `border-b-2 border-primary` (sky blue underline) — detected via `usePathname()` exact or prefix match.
- Active matching rules:
  - `/` → exact match only (Catalog)
  - `/collection` → prefix match (Collection)
  - `/decks` → prefix match (Decks, covers `/decks/[id]` as well)
- Mobile: Links remain visible (three short labels fit at mobile widths). No hamburger menu needed.
- Nav bar padding: `px-md` (16px) mobile, `px-xl` (32px) desktop (`md:px-xl`).
- Gap between nav links: `sm` (8px) via `gap-2` or `gap-sm`.

### 2. Per-Deck Want List Tab (deck builder toolbar + `WantListTab` component)

Source of decisions: D-01, D-02, D-03, D-04 (05-CONTEXT.md)

**Tab group extension:** The existing two-button tab group in `deck-builder.tsx` gains a third button:

```
[ Add Cards ]  [ Deck List ]  [ Want List ]
```

- Container: existing `flex bg-slate-100 rounded-lg p-1` pill group — no styling change.
- Active state: `bg-white shadow-sm` (matching existing pattern from `deck-builder.tsx` lines 218–232).
- Inactive state: `variant="ghost"`, `size="sm"`.
- New `view` state value: `'want-list'` added to `'editor' | 'catalog'` union.

**Want List tab content (`WantListTab` component):**

Rendered in the same `flex-1 overflow-y-auto bg-slate-50` scroll area used by catalog and editor views.

Layout: `max-w-4xl mx-auto p-lg` (24px padding). Cards grouped by type with a section header per group.

Type group order (D-03): Leader → Base → Unit → Event → Upgrade.

Type group header: Label (12px, semibold, Nunito Sans), `text-muted-foreground uppercase tracking-wider`, `mb-sm` (8px below), `border-b border-muted pb-xs` (4px below label, muted rule line).

Card grid within each group: CSS grid, responsive columns matching the existing catalog/collection grid breakpoints:

| Breakpoint | Columns |
|------------|---------|
| < 640px (mobile) | 3 |
| 640–1023px (sm) | 5 |
| 1024–1279px (md) | 7 |
| >= 1280px (lg) | 9 |

Gap: `sm` (8px) all breakpoints — matching existing `card-grid.tsx`.

**Want List tile (`WantListCardItem` component — new, extends `CardItem` visual language):**

Reuses `CardItem` structure (image aspect ratio, placeholder, hover ring) but in a new read-only `'want-list'` mode:

- No +/− controls in hover overlay (read-only — D-02).
- No collection controls.
- Hover overlay shows card name only (for accessibility on small tiles): `text-white text-xs font-semibold text-center px-1` centered in overlay.
- Card tile border: `border-red-500 shadow-[0_0_10px_rgba(239,68,68,0.5)]` (shortfall > 0 always true since only shortfall cards are shown).
- Three stat chips below the card image (always visible, not hover-only):
  - **NEED** chip: `bg-muted text-foreground text-[9px] font-bold px-1.5 py-0.5 rounded-sm` — shows `quantity` (deck quantity needed).
  - **OWN** chip: same styling — shows `ownedCount`.
  - **SHORT** chip: `bg-destructive text-destructive-foreground text-[9px] font-bold px-1.5 py-0.5 rounded-sm` — shows shortfall count (need − owned).
- Chip row: `flex gap-xs justify-center mt-xs` (4px gap, 4px top margin).
- Card name: NOT rendered below tile (too small at grid scale). Accessible via `alt` on the image and `aria-label` on the container.

### 3. Combined Want List Section (`/decks` dashboard, `src/app/decks/page.tsx`)

Source of decisions: D-05, D-06, D-07, D-08 (05-CONTEXT.md)

**Section position:** Below the existing deck list grid. Separated by `mt-2xl` (48px) top margin.

**Section heading:** "What I Need to Buy" — Heading style (20px Oxanium semibold), `mb-sm` (8px below heading).

**Summary line:** Rendered directly below the heading, above the card grid.

Format: `"{N} cards needed, {M} total copies short"` — Body (14px Nunito Sans weight 400), `text-muted-foreground`, `mb-lg` (24px below before card grid).

Example: `"11 cards needed, 23 total copies short"`.

**Card grid:** Same responsive column breakpoints as per-deck want list (3/5/7/9 columns). Gap: `sm` (8px). Cards grouped by type (Leader → Base → Unit → Event → Upgrade) with same type group headers.

**Combined want list tile:** Same `WantListCardItem` visual as per-deck want list. Stat chips show:
- **NEED** chip: max quantity needed across any deck.
- **OWN** chip: owned count.
- **SHORT** chip: shortfall (max_needed − owned).

**Loading state:** Summary line shows "Calculating..." in `text-muted-foreground` while data is being fetched. Card grid shows a row of 5 `bg-muted animate-pulse rounded-md aspect-[2/3]` placeholders.

**Section only shown when:** At least one deck exists AND at least one card has shortfall > 0. If user has no shortfall across any deck, section is hidden entirely (no empty state shown for this section — silence is the correct message when everything is owned).

## Interaction Contracts

### Tab Switching (deck builder)

- Tab buttons are `<Button>` components (`variant="ghost" | "secondary"`, `size="sm"`).
- Switching tabs does NOT reset scroll position of the scroll area (each view manages its own scroll offset independently).
- When switching to "Want List" tab for the first time, collection data is fetched via `GET /api/collection` (same pattern as `CatalogClient`). Subsequent tab switches use cached local state — no re-fetch unless component unmounts.
- Loading state for want list tab: full-area centered spinner — `<div className="flex items-center justify-center h-64"><p className="text-muted-foreground">Loading want list...</p></div>`.

### Shortfall Badge Behavior

- A card appears in the want list tile grid ONLY when `shortfall > 0` (need − owned > 0). Zero-shortfall cards are omitted.
- Shortfall badge (`SHORT` chip) background: `bg-destructive` (red) — always visible below tile, not hover-gated.
- Badge text: numeric shortfall only (e.g. `2`). No unit label on the badge itself (label "SHORT" is the chip prefix).
- If shortfall is ≥ 10, badge uses same font size — no truncation.

### Active Nav Link

- Active link detection uses `usePathname()` from `next/navigation` (client component required for nav bar).
- Active indicator: `border-b-2 border-primary` bottom border on the link element, `text-foreground font-semibold`.
- Transition: `transition-colors duration-150` on hover state change.
- No active indicator on hover (only on actual active route).

### Empty States

- **Want List tab — no shortfall cards:** Show centered empty state inside the tab scroll area. See Copywriting Contract below.
- **Combined want list section — no shortfall:** Section is entirely hidden. No empty state rendered.
- **Want List tab — deck has no cards:** Show empty state prompting user to add cards first. See Copywriting Contract below.

### Image Loading (want list tiles)

Inherits Phase 2/3/4 pattern: `bg-muted animate-pulse` placeholder until `onLoad` fires, then opacity transition to image. `onError` stops pulse, leaves placeholder.

## Implementation Notes for Executor

These are precise contracts, not suggestions.

1. **`NavBar` component location:** `src/components/nav-bar.tsx`. Imported in `src/app/layout.tsx` inside `<body>` as a sibling above `<NuqsAdapter>`. Must be `'use client'` for `usePathname()`.

2. **Layout height adjustment:** The deck builder currently uses `h-[calc(100vh-64px)]` — this assumes a 64px top bar. The nav bar is 56px (h-14). Adjust to `h-[calc(100vh-56px)]` or use `h-[calc(100svh-56px)]` for mobile browser chrome safety. Confirm and update the calc value.

3. **`WantListTab` component location:** `src/components/decks/want-list-tab.tsx`. Receives `deckCards: { cardDefinitionId: number; quantity: number }[]` and computes shortfall against `GET /api/collection` internally.

4. **`WantListCardItem` component:** New `mode="want-list"` prop value on existing `CardItem`, OR a separate `WantListCardItem` component in `src/components/catalog/want-list-card-item.tsx`. Prefer extending `CardItem` with a `mode="want-list"` branch to avoid duplicating image loading logic.

5. **Tile stat chips:** Rendered as `<div className="flex gap-1 justify-center mt-1">` below the card image container (`div.group`), outside the `<Link>` element. Not inside the hover overlay.

6. **Combined want list data fetch:** The `/decks` dashboard currently fetches only deck metadata. For the combined want list, either: (a) convert the dashboard to a Server Component and fetch all deck cards server-side, or (b) add a new API route `GET /api/want-list` that returns the aggregated shortfall data. Prefer (b) for consistency with the existing client-component fetch pattern in the deck builder.

7. **Type group ordering:** Hard-coded sort order array `['Leader', 'Base', 'Unit', 'Event', 'Upgrade']`. Cards with types not in this array sort last, alphabetically.

8. **Nav bar active route for `/`:** Use `pathname === '/'` (exact match). All other routes use `pathname.startsWith('/collection')` and `pathname.startsWith('/decks')`.

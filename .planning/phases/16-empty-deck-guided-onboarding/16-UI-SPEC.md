## New Component Contracts

### Auto-Filter Chip (D-09, D-10)

The chip is the only net-new visual element in this phase. It uses the existing `Badge` component with a custom `className` override — no new variants are added to badge.tsx.

#### Rendering rule

| Deck state | `isAutoFilterOverridden` | Chip text | Chip visible |
|------------|--------------------------|-----------|--------------|
| No leader AND no base | false | `Auto: Leader & Base` | Yes |
| Leader only, no base | false | `Auto: Leader & Base` | Yes |
| Both leader AND base selected | false | `Auto: Aspect filter` | Yes |
| Any state | true | — | No (not rendered) |

When `isAutoFilterOverridden` becomes `true`, the chip is removed from the DOM entirely (conditional render, not `visibility: hidden`).

#### Exact JSX contract

```tsx
<Badge
  variant="outline"
  className="border-primary/40 bg-primary/10 text-primary dark:border-primary/30 dark:bg-primary/15 dark:text-primary-foreground"
  aria-label={`Auto-filter active: ${autoFilterLabel}`}
  role="status"
>
  {autoFilterLabel}
</Badge>
```

- `variant="outline"` provides the base border + transparent-bg structure; the `className` overrides border color, background, and text to the primary (blue) family.
- Text color: `text-primary` (light) / `text-primary-foreground` is not correct here — use `text-primary` in both modes since primary darkens but stays within blue family. Dark mode override: `dark:text-primary` resolves to `oklch(0.443 0.11 240.79)` which is readable on `dark:bg-primary/15`.
- Badge height: `h-5` (20px) — badge component default, do not override.
- Badge font: `text-xs font-medium` — badge component default, do not override.
- Badge radius: `rounded-4xl` — badge component default, do not override.
- No icon. No dismiss control. No click handler. The element is presentation-only.

#### Accessibility

- `role="status"` — announces chip text changes to screen readers without interrupting the user.
- `aria-label` — provides explicit description: `"Auto-filter active: Auto: Leader & Base"` or `"Auto-filter active: Auto: Aspect filter"`.
- When chip is removed (overridden), the DOM node is absent — screen readers receive no announcement for removal (intentional: silent disappearance on user action).

#### Placement in SidebarFilters (D-10)

Position: immediately below the `<h2>Filters</h2>` heading, above the search `<Input>`.

Exact insertion point in sidebar-filters.tsx layout:

```
<h2>Filters</h2>           ← existing, line 84
[AUTO-FILTER CHIP HERE]    ← new, conditional
<Input search />           ← existing, line 87
<Owned-only switch>
...
```

Spacing: no wrapper div needed. The sidebar's existing `flex flex-col gap-4` provides 16px gap above and below the chip automatically. The chip is self-contained as a single inline-flex element.

#### Prop added to SidebarFilters

```ts
autoFilterLabel?: string | null  // undefined or null = chip not rendered
```

- When `autoFilterLabel` is a non-empty string, the chip renders.
- When `autoFilterLabel` is `null`, `undefined`, or empty string, the chip is not rendered.
- `MobileFilterSheet` passes all `SidebarFilters` props via `{...props}` (line 35 of mobile-filter-sheet.tsx) — the new prop propagates automatically with no change to MobileFilterSheet.

#### Dark mode

- Light: `border-primary/40 bg-primary/10 text-primary` — subtle blue tint on white sidebar.
- Dark: `dark:border-primary/30 dark:bg-primary/15 dark:text-primary` — same hue family, slightly lower opacity to avoid over-saturation on dark card bg (`oklch(0.21)`).

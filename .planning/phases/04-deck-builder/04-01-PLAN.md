<task type="auto">
  <name>Task 1: Define Decks and DeckCards schema with draft support</name>
  <files>src/db/schema.ts</files>
  <action>
    Add `decks` and `deck_cards` tables to the Drizzle schema.
    - `decks`: 
      - `id`: serial PK
      - `userId`: integer, default 1
      - `name`: text, not null
      - `leaderCardDefinitionId`: integer, references card_definitions(id)
      - `baseCardDefinitionId`: integer, references card_definitions(id)
      - `isDraft`: boolean, default true (per D-01)
      - `createdAt`, `updatedAt`: timestamps
    - `deck_cards`: 
      - `deckId`: integer, ref decks(id), cascade delete
      - `cardDefinitionId`: integer, ref card_definitions(id)
      - `quantity`: integer
      - `isSideboard`: boolean, default false
    - Composite PK for `deck_cards`: [deckId, cardDefinitionId, isSideboard].
  </action>
  <verify>
    <automated>npx drizzle-kit generate</automated>
  </verify>
  <done>Schema defined with `isDraft` column and migrations generated.</done>
</task>

<task type="auto">
  <name>Task 2: Implement Deck query functions</name>
  <files>src/db/queries/decks.ts</files>
  <action>
    Create data access functions for decks, using `cardDefinitionId` consistently:
    - `getDecks(userId: number)`: Returns all decks for a user.
    - `getDeckWithCards(deckId: number)`: Returns a deck and all its associated cards.
    - `createDeck(name: string, userId: number)`: Creates an empty deck shell (isDraft: true).
    - `updateDeck(deckId: number, data: { name?: string, leaderCardDefinitionId?: number, baseCardDefinitionId?: number, isDraft?: boolean, cards: { cardDefinitionId: number, quantity: number, isSideboard: boolean }[] })`:
      Updates deck metadata and replaces ALL cards in a transaction (delete + insert).
    - `deleteDeck(deckId: number)`: Deletes a deck.
  </action>
  <verify>
    <automated>npm run check-db</automated>
  </verify>
  <done>Query functions implemented and handle card updates atomically using standard naming.</done>
</task>

<task type="auto">

  <files>src/app/api/decks/route.ts, src/app/api/decks/[id]/route.ts</files>
  <action>
    Implement Next.js App Router API handlers:
    - `GET /api/decks`: List all decks for userId 1.
    - `POST /api/decks`: Create a new deck (body: { name }).
    - `GET /api/decks/[id]`: Get detailed deck with cards.
    - `PATCH /api/decks/[id]`: Update deck (body: { name, leaderCardDefinitionId, baseCardDefinitionId, isDraft, cards }).

    Ensure `await params` is used for `id`.

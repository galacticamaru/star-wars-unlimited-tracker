# Phase 5.2 Discussion Log

## 2026-05-07: Handling Rarity Value Discrepancy

**Observation**: 
The UI labels in `catalog-client.tsx` are formatted as `(C) Common`, `(U) Uncommon`, etc., while the database and the internal `CardForFilter` interface use simple values like `Common`, `Uncommon`.

**Options**:
1. Change `RARITY_OPTIONS` to simple values and update `FilterDropdown` to handle labels separately.
2. Map the values in `filterCards` to strip the prefix.
3. Update `catalog-client.tsx` to pass mapped values to `filterCards`.

**Decision**:
Option 2: Map the values in `filterCards`. This is the most robust approach as it keeps the UI logic separate from the filtering logic, and `filterCards` can easily handle both simple values (for tests) and UI-formatted values (from the live app) by checking for the space/prefix.

**Logic**:
```typescript
const matchesRarity = (selectedRarities?.length ?? 0) === 0 ||
  selectedRarities.some(r => {
    // If r is "(C) Common", extract "Common"
    // If r is "Common", use as is
    const simpleRarity = r.includes(' ') ? r.split(' ').slice(1).join(' ') : r;
    return simpleRarity === card.rarity;
  });
```

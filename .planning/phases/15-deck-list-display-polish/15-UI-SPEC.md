## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding |
| sm | 8px | Compact element spacing (button gaps, row inner padding) |
| md | 16px | Default element spacing (card row padding p-4, sidebar inner padding) |
| lg | 24px | Section padding (editor view p-6, space between type-group sections) |

## Layout Contract

### Deck List Tab (editor view) — updated layout

```
[Preview Panel (hidden when none hovered)] | [Main content: leader/base grid + type-group card list]
```

- Layout: `flex flex-row gap-6` wrapping the preview panel and the main card list content
- Preview panel: `w-48 shrink-0` (192px wide), `sticky top-0`, hidden when no card is hovered (`invisible` or conditional render)
- Preview panel shows `<Image src={hoveredCard.frontArtUrl} fill object-cover />` inside a `aspect-[2/3] rounded-lg overflow-hidden` container
- On screens `< md` (< 768px): preview panel does not float to the left — instead, a fixed bottom bar 96px tall with the card image appears on tap/focus (mobile-safe)

### Type-Group Sections (within main content)

Render order: Ground Units → Space Units → Upgrades → Events → Other (catch-all, hidden if empty)

Each group section:
```
<section>
  <h3 class="text-sm font-semibold uppercase text-slate-500 mb-2">
    Ground Units ({count})
  </h3>
  <div class="bg-white border rounded-lg divide-y shadow-sm">
    {card rows}
  </div>
</section>
```

Spacing between groups: `space-y-6` (24px)

### Leader / Base Art Slot (filled state)

- Container: same `aspect-[3/4]` (leader) / `aspect-[4/3]` (base) border-2 border-dashed rounded-lg overflow-hidden — unchanged dimensions
- Content: `<Image src={frontArtUrl} alt={card.name} fill sizes="..." className="object-cover" />`
- Remove overlay: `absolute inset-0 opacity-0 group-hover:opacity-100 transition-opacity bg-black/40 flex items-center justify-center` containing a `<Button variant="ghost" size="sm" className="text-white bg-black/30 hover:bg-black/50 border border-white/20">Remove</Button>`
- Wrapper must be `group relative` to trigger the hover pattern

### Aspect Breakdown Panel (deck-sidebar.tsx)

Position: after the Types/Arenas grid, before the Save buttons section

```
<div>
  <h3 class="text-sm font-semibold uppercase text-slate-500 mb-2">Aspects</h3>
  <div class="space-y-1">
    {Object.entries(aspectCounts)
      .filter(([aspect]) => aspect !== 'Basic')
      .sort(([,a],[,b]) => b - a)
      .map(([aspect, count]) => (
        <div class="flex justify-between text-sm">
          <span class="text-slate-600">{aspect}</span>
          <span class="font-medium">{count}</span>
        </div>
    ))}
  </div>
</div>
```

Sorted by count descending so the dominant aspect appears first.

## Interaction Contract

### Hover Art Preview

| Trigger | Platform | Behavior |
|---------|----------|----------|
| `mouseenter` on card row | Desktop | Show preview panel with that card's `frontArtUrl`; update `hoveredCard` state |
| `mouseleave` on card row | Desktop | Clear `hoveredCard` state; preview panel returns to empty/hidden state |
| `focus` on card row (keyboard) | All | Same as mouseenter — show preview |
| `blur` on card row (keyboard) | All | Same as mouseleave — hide preview |
| `touchstart` / tap on card row | Mobile | Toggle fixed-bottom preview bar with card art |

Preview panel transition: `transition-opacity duration-150` — fade in/out, not slide

### Leader/Base Art Remove Button

| Trigger | Behavior |
|---------|----------|
| `mouseenter` on filled art slot | Overlay fades in (opacity-0 → opacity-100) |
| Click "Remove" in overlay | Dispatches `SET_LEADER: null` or `SET_BASE: null`; slot reverts to dashed placeholder |
| `mouseleave` | Overlay fades out |

No confirmation dialog required — removing leader/base is easily reversible.

### Empty Sections

Type-group sections with 0 cards are not rendered at all (D-02). The heading is NOT shown.

## Accessibility Notes

- Leader/Base art slot: `alt={card.name}` on the `<Image>` element
- Preview panel: `aria-live="polite"` on the preview container so screen readers announce when card art changes
- Remove overlay button: `aria-label="Remove {card.name} as leader"` / `aria-label="Remove {card.name} as base"`
- Type-group sections: use `<section>` with an `aria-label` matching the heading text (e.g. `aria-label="Ground Units"`)
- Hover state on card rows: add `tabIndex={0}` so keyboard users can trigger the art preview

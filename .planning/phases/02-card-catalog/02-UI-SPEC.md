## Layout Contracts

### Catalog Page (`/` or `/catalog`)

**Top Bar** (D-03): full-width sticky bar, height 56px, `bg-background/80 backdrop-blur-sm`, `border-b border-border`.

Structure (left to right, single row):
1. Search input — `flex-1`, minimum 200px, maximum 480px, placeholder "Search cards..."
2. Set filter dropdown — fixed width 120px
3. Type filter dropdown — fixed width 120px
4. Aspect filter dropdown — fixed width 120px

Top bar padding: `px-md` (16px) mobile, `px-xl` (32px) desktop. Gap between elements: `sm` (8px).

**Card Grid** (D-01, D-04):

Full render — all 1,806 non-token cards loaded and rendered on page load. No pagination, no infinite scroll. Client-side filtering via `useMemo` over the full card array.

Grid column count by breakpoint (responsive CSS grid):

| Breakpoint | Columns | Card width (approx) |
|------------|---------|---------------------|
| < 640px (mobile) | 3 | ~33vw |
| 640–1023px (sm) | 5 | ~180px |
| 1024–1279px (md) | 7 | ~136px |
| 1280–1535px (lg) | 9 | ~130px |
| >= 1536px (xl) | 11 | ~128px |

Grid gap: `sm` (8px) all breakpoints.
Card container: `aspect-[2/3]` (D-02 — SWU portrait ratio). `rounded-md overflow-hidden`. No border, no shadow.

**Card placeholder** (D-02): `bg-muted animate-pulse rounded-md aspect-[2/3]`. No broken-image icon on error — placeholder remains.

**Card image**: `<Image>` from `next/image`. `fill` layout within the `aspect-[2/3]` container. `sizes` attribute tuned per breakpoint column count. `object-fit: cover`. On hover: `ring-2 ring-primary ring-offset-1 cursor-pointer transition-shadow`.

**Result count**: Single line above the grid, right-aligned. Format: "1,806 cards" / "143 cards" (filtered count). Font: Label (12px, semibold). Color: `text-muted-foreground`.

### Card Detail Page (`/cards/[set-code]/[card-number]`)

**Layout** (D-07): Two-column side-by-side, `max-w-5xl mx-auto px-md py-2xl`.

| Column | Width | Content |
|--------|-------|---------|
| Left | 320px fixed | Full card image, `aspect-[2/3]`, `rounded-lg`, no border |
| Right | `flex-1` | Metadata panel |

On mobile (< 768px): stack vertically — image full width, metadata below.

**Metadata panel** fields (in display order):
1. Card name — Heading (20px Oxanium semibold)
2. Subtitle — Body (14px italic, `text-muted-foreground`)
3. Type / Arenas / Aspects — Label row (12px semibold, flex-wrap)
4. Traits / Keywords — Body (14px), comma-separated
5. Cost / Power / HP — three inline stat chips
6. Front text (abilities) — Body (14px, `bg-muted rounded-md p-md`)
7. Back text (if double-sided) — same as front text, labelled "Back"
8. Epic Action (if Leader) — Body (14px, `bg-muted rounded-md p-md`), labelled "Epic Action"
9. Set / Collector Number / Rarity / Artist — Label (12px, `text-muted-foreground`)

**Back button**: top-left, `variant="ghost"` Button with ChevronLeft icon. Label: "Back to catalog". Routes to `/`.

## Interaction Contracts

### Search (D-04)

- Input type: `text`, controlled, `'use client'` component
- Debounce: none — filter on every keystroke (`onChange`). 1,806 cards × simple string match is <5ms.
- Match strategy: case-insensitive substring match on `card_definitions.name`
- Clear button: appears when input has value. X icon (`lucide: X`), `variant="ghost"` icon-only button, 36px square, inside the input trailing slot.

### Filters (D-03, D-05)

- Each filter is a shadcn `<DropdownMenu>` with checkboxes (multi-select per category).
- Trigger shows category name + active count badge when filters are selected. Example: "Set (2)".
- AND logic across categories (D-05): each active filter from Set, Type, Aspect independently narrows the result set.
- Filter options are populated from distinct values queried from the DB at page load (Server Component passes options as props to the client filter component). No hard-coded option lists.
- Token cards are excluded from the catalog query (`type NOT ILIKE '%token%'` or T-prefixed collector number exclusion per Phase 1 convention).

### Card Click

- Entire card container is a Next.js `<Link href="/cards/{setCode}/{cardNumber}">` wrapping the image.
- No JavaScript `onClick` — native link navigation.

### Image Loading

- `next/image` with `placeholder="blur"` is not used (no blur data URL available from the API).
- Instead: render the grey `bg-muted animate-pulse` placeholder as the card container background. The `<Image>` renders on top once loaded. `onLoad` removes the pulse class. `onError` leaves the placeholder visible with pulse stopped.

## Next.js Image Configuration

Per D-08. Add to `next.config.ts`:

```ts
images: {
  remotePatterns: [
    {
      protocol: 'https',
      hostname: 'cdn.swu-db.com',
      pathname: '/**',
    },
  ],
}
```

The resolved CDN hostname is `cdn.swu-db.com` (confirmed from `front_art_url` values in `card_printings`).
